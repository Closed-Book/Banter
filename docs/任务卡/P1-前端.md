# P1 · 前端主力 任务卡

**独占目录**：`src/`（只在这里写文件）
**只读依赖**：`shared/contract.ts`、`shared/mock-responses/`、`docs/API契约.md`
**核心产出**：可交互的完整五屏应用

---

## 你要交付的五屏

| 屏 | 路由 | 内容 |
|----|------|------|
| 1 场景选择 | `/` | 三级标签逐层点选；MVP 场景高亮，其余置灰+锁标；底部轮次选择(8/10/15，15 锁) |
| 2 人设配置 | `/persona` | 快速问答模式(3 题) / 跳过用默认人设；答案拼接注入 system prompt |
| 3 Briefing | `/briefing` | 剧本杀开场卡：场景描述 + 目标 + 对手简介 + 轮次提示；"进入副本"按钮 |
| 4 对话主界面 | `/battle` | 顶部 3 数值条 + 中部聊天气泡区 + 底部 4 张卡牌区 |
| 5 结局评分 | `/ending` | 结局等级 + 点评 + 五维雷达图(recharts) + 最佳/最危险一句话回放 + 行动建议 |

## 第四屏单轮循环（核心）

1. NPC 发言（气泡逐字打出，多人则按 `npc_responses` 顺序间隔 0.5s 弹出）
2. 底部展开 4 张卡牌
3. 点卡牌 → 展开看完整话术 → 确认出牌 → 用户气泡出现
4. 数值条 CSS transition 动画变化（按 `stats_change`）
5. NPC 回应 → 下一轮
6. `game_state.is_ended=true` → 跳结局页

## 必须做到的工程点

- **零计算渲染**：拿到 JSON 直接画，不在前端算数值。
- **API client**（`src/api/`）：
  - 封装对阶跃星辰的请求（OpenAI 兼容，`response_format: json_object`）。
  - 读 `import.meta.env.VITE_USE_MOCK`，为 `true` 时返回 `shared/mock-responses/` 的假数据（**让你不依赖 P2 就能跑通五屏**）。
  - 容错：JSON.parse 失败自动重试一次；连续两次失败展示"网络波动"5 秒后重试。
- **状态管理**（`src/state/`）：当前场景/人设/轮次/数值/对话历史。用 Context 或简单 store 即可，别上 redux。
- import 契约常量：`import { CARD_TYPES, DEFAULT_STATS, STAT_LABELS, RADAR_LABELS } from '@shared/contract'`

## 时间线（对齐手册 · 六）

- **第 1 小时**：项目跑起来、第 1/2 屏 UI、和 P2 对齐接口（直接以 `contract.ts` 为准）。
- **第 2 小时**：第 3/4 屏布局，接 mock 跑通单轮循环，第 5 屏布局。
- **第 3 小时**：接 P2 真实 API、修 P4 反馈的 bug、打磨动效（数值条/打字机/卡牌展开）。
