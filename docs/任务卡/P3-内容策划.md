## scenarios.json 结构（供 P1 直接读）

三级标签：一级场景大类(职场/家庭/学校/社交) → 二级具体事件 → 三级对手性格(可叠加)。
每个二级事件标注 `playable: true/false`（MVP 只有两个 true，其余 UI 展示"即将开放"）。
MVP 锁定：职场×老板PUA压薪×笑面虎(1v1) + 职场×同事甩锅×暴躁同事+观望领导(多人)。
建议字段：

```jsonc
{
  "categories": [
    {
      "id": "workplace", "name": "职场", "keywords": ["老板","同事","HR","甲方"],
      "events": [
        { "id": "boss_pua_salary", "name": "老板PUA你压薪", "playable": true, "multiplayer": false },
        { "id": "colleague_blame", "name": "同事甩锅到你头上", "playable": true, "multiplayer": true },
        { "id": "hr_ghosting", "name": "HR已读不回要跟进", "playable": false }
      ]
    }
  ],
  "personalities": [
    { "id": "smiling_tiger", "name": "笑面虎型", "feature": "表面温和，句句埋刀" }
  ]
}
```

全量标签清单见手册 · 二（一级 4 类、二级各事件、三级 5 种性格）。

## role-behavior.md（最关键，交给 P2）

定义老板/同事/领导在什么条件下：让步 / 加压 / 转移话题 / 摊牌。
10 轮节奏曲线：轮 1-3 试探、轮 4-7 拉锯、轮 8-10 摊牌。
这份清单是 P2 写"不可违反规则"和"数值规则"的依据，**第一小时和 P2 结对产出**。

## briefing 文案（剧本杀开场卡）

每个可玩场景一张，含：场景描述(你是谁/在哪/发生什么) + 你的目标(明确通关条件) + 对手简介(性格标签不剧透行为) + 轮次提示。
场景 1 示例见手册 · 三第三屏（📍会议室 / 👤入职两年设计师 / 🎯涨薪≥10% / ⚠️张总笑面虎 / ⏱10 轮）。

## scoring.md（结局评分）

5 个等级判定 + 一句话点评模板（对齐 `contract.ts` 的 `GRADE_RULES`）：

| 等级 | 条件 | 关键词 |
|------|------|--------|
| S | 目标达成率≥90 且 主动权≥70 | 高情商通关 |
| A | 目标达成率≥70 | 稳定通关 |
| B | 目标达成率≥50 | 勉强过关 |
| C | 目标达成率<50 且 耐心>0 | 被拿捏了 |
| D | 耐心归零(谈崩) | 把天聊死了 |

每个等级写 2-3 句可复用的点评文案 + 现实行动建议风格示范。

## persona-qa.md（人设问答 3 题）

对齐手册 · 三第二屏：①你的身份？②冲突中你通常？③你最怕对方说什么？各给 4 个选项 + 自定义。说明答案如何拼接成人设描述注入 prompt。

## 时间线

- **第 1 小时**（前半与 P2 结对）：role-behavior.md(行为边界+节奏曲线) → 场景 1 briefing → scoring.md 5 级判定+点评。
- **第 2 小时**：其余不可玩场景的展示文案、三级标签性格描述、人设 Q&A 文案；辅助 P2 调对话质量。
- **第 3 小时**：Demo 引导文案(评委首次进来的引导语) + 1 分钟 pitch 话术(见手册 · 九)。
