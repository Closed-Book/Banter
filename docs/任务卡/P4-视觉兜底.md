## tokens.css（最优先，P1 等着用）

第一小时就把设计 token 给 P1。建议：

```css
:root {
  /* 配色 */
  --color-bg: #0f1115;
  --color-surface: #1a1d24;
  --color-primary: #6c5ce7;
  --color-text: #e8e8ea;
  /* 四类卡牌色（对应 contract.ts 的 CardType） */
  --card-high-eq: #00b894;        /* 高情商 */
  --card-veteran: #fdcb6e;        /* 老油条 */
  --card-naive: #b2bec3;          /* 学生思维 */
  --card-confrontational: #e17055;/* 硬刚 */
  /* 三数值条色 */
  --stat-initiative: #74b9ff;     /* 主动权 */
  --stat-patience: #fab1a0;       /* 对方耐心 */
  --stat-goal: #55efc4;           /* 目标达成率 */
  /* 字号 / 间距 / 圆角 */
  --font-lg: 20px; --font-md: 15px; --font-sm: 13px;
  --space-1: 4px; --space-2: 8px; --space-3: 16px; --space-4: 24px;
  --radius: 12px;
}
```

颜色键务必和 `shared/contract.ts` 的 `CARD_TYPES`(high_eq/veteran/naive/confrontational) 与 `STAT_KEYS`(initiative/patience/goal_progress) 对应，P1 才能按类型上色。

## 动效方案（写进 style-guide.md，P1 实现）

- 数值条：`transition: width 0.6s ease-out`，变化时颜色微闪。
- 气泡：逐字打字机效果，多人间隔 0.5s 弹出。
- 卡牌：点击展开看完整话术的展开动画。

## 兜底职责（手册 · 八，你是 Demo 生死线守门人）

- **录屏备份完整通关过程**：网络挂了用录屏应急。准备 4G 热点。
- **测试清单.md**：手动走完五屏，记录每个 bug + UI 问题，实时反馈给 P1。
- **demo脚本.md**：评委首次进来的引导、1 分钟 pitch 配合(话术见手册 · 九)、最终 5 分钟全员围观你跑一遍完整 demo 确认无阻断 bug。

## 时间线

- **第 1 小时**：定视觉方案(配色/卡牌/数值条) → 找/做素材 → 数值条动效参数 → **token 交给 P1**。
- **第 2 小时**：素材+样式交付 P1、协助写 CSS；第一轮自测(场景选择→人设→briefing)，记 bug 反馈 P1。
- **第 3 小时**：完整走 demo 模拟评委视角、录屏备份、查所有文案错别字、最终全员围观跑通。
