# 《别把天聊死》Hackathon 项目执行手册

> AI现实社交副本训练智能体 — 3小时极限冲刺版

---

## 一、产品一句话定义

用剧本杀的形式训练真实社交能力：选场景、选对手、出牌对话、数值博弈、通关评分。
不是话术生成器，是让你开口练的实战沙盘。

---

## 二、场景标签体系

三级标签，用户逐层选择进入副本。

### 一级：场景大类

| 场景 | 关键词 |
|------|--------|
| 职场 | 老板、同事、HR、甲方 |
| 家庭 | 父母、亲戚、伴侣 |
| 学校 | 导师、室友、辅导员 |
| 社交 | 朋友、饭局、陌生人 |

### 二级：具体事件

**职场**
- 老板PUA你压薪（MVP首发 ✅）
- 同事甩锅到你头上（MVP第二场景 ✅，多人）
- HR已读不回要跟进
- 领导画饼拒绝加薪
- 被迫加班要说不

**家庭**
- 催婚催生逼问
- 亲戚饭桌比较"别人家的孩子"
- 父母干涉职业选择
- 家庭开支分配谈崩

**学校**
- 导师办公室谈话
- 小组作业有人摆烂
- 室友生活习惯冲突
- 被辅导员约谈

**社交**
- 朋友开口借钱
- 被道德绑架帮忙
- 饭局上被长辈灌酒劝酒

### 三级：对手性格修饰（通用，可叠加到任何二级事件）

| 类型 | 特征 | Prompt锚点 |
|------|------|-----------|
| 沉着冷静型 | 不急不躁，用沉默施压 | 语速慢，停顿多，反问为主 |
| 暴躁易怒型 | 容易拍桌子，声量压人 | 短句，感叹号多，打断用户 |
| 笑面虎型 | 表面温和，句句埋刀 | 语气亲切但每句都在施压，善用"我也是为你好" |
| 情感绑架型 | 用关系/恩情/道德绑架 | "我当初怎么带你的""你忍心吗" |
| 逻辑碾压型 | 用数据/事实/制度堵你的嘴 | 引用规章、数据、先例，让你无从反驳 |

**MVP锁定**：职场×老板PUA压薪×笑面虎型（1v1）+ 职场×同事甩锅×暴躁同事+观望领导（多人）。
其余场景UI展示标签和briefing文案，标注"即将开放"。

## 三、核心游戏流程

共五屏，用户线性推进。

### 第一屏：场景选择
三级标签逐层点选。MVP可玩场景高亮，其余置灰+锁标。
底部：轮次选择（8轮速战 / 10轮标准 ⭐默认 / 15轮深度🔒即将开放）

### 第二屏：人设配置（可跳过）
两种模式切换：

**快速问答模式（推荐）**：3个问题蒸馏用户人设
1. 你的身份是？→ 实习生 / 刚入职 / 老员工 / 小主管
2. 冲突中你通常？→ 据理力争 / 先忍再说 / 找人帮忙 / 冷处理
3. 你最怕对方说什么？→ "你还年轻" / "大家都这样" / "你考虑下后果" / 自定义输入

答案拼接为人设描述，注入system prompt。

**跳过** → 使用默认人设：入职两年普通员工，性格中等偏内向。

### 第三屏：场景Briefing
剧本杀式开场卡，包含：
- 场景描述（你是谁、在哪、发生了什么）
- 你的目标（明确的通关条件）
- 对手简介（性格标签，不剧透行为模式）
- 轮次提示（"你有10轮对话机会"）

示例（MVP场景1）：
> 📍 会议室，下午3点，只有你和老板
> 👤 你是入职两年的设计师，月薪8K
> 🎯 年终review，老板约你单聊。你的目标：拿到至少10%涨薪，不被PUA打回去
> ⚠️ 对手：张总，笑面虎型，表面和善但绝不轻易让步
> ⏱ 10轮对话，请谨慎出牌

点击"进入副本"开始。

### 第四屏：对话主界面

**布局**：
- 顶部：三个数值条（主动权 / 对方耐心 / 目标达成率），实时变化
- 中部：聊天气泡区（NPC头像+对话内容，多人场景多个头像）
- 底部：四张卡牌区

**单轮循环**：
1. NPC发言（气泡逐字打出）
2. 底部展开4张卡牌
3. 用户点击卡牌 → 展开查看完整话术 → 确认出牌
4. 用户话术以气泡形式出现
5. 数值条动画变化
6. NPC回应 → 进入下一轮

**四张卡牌的策略类型（固定）**：

| 卡牌 | 类型 | 风格说明 |
|------|------|---------|
| A | 高情商 | 认可对方但不让步，以退为进 |
| B | 老油条 | 打太极、拖时间、转移焦点 |
| C | 学生思维 | 老实听话、容易被拿捏、表态模糊 |
| D | 硬刚 | 直接怼，摆事实讲道理，不留面子 |

每张卡牌展示：策略标签 + 一句话摘要（前端可见）+ 完整话术（点击展开可见，作为实际对话内容发送）。

**多人场景补充规则**：
- 气泡带不同头像和姓名标识
- NPC按剧本顺序逐条出现（间隔0.5s打字机效果）
- 卡牌可标注对话对象（"对同事说" / "对领导说"）

### 第五屏：结局与评分

**结局等级判定**（基于最终数值）：

| 等级 | 条件 | 关键词 |
|------|------|--------|
| S | 目标达成率≥90% + 主动权≥70% | 高情商通关 |
| A | 目标达成率≥70% | 稳定通关 |
| B | 目标达成率≥50% | 勉强过关 |
| C | 目标达成率<50% + 耐心>0 | 被拿捏了 |
| D | 耐心归零（谈崩） | 把天聊死了 |

**评分报告内容**：
- 结局等级 + 一句话点评
- 五维能力雷达图（表达力 / 共情力 / 策略性 / 抗压力 / 边界感）
- 最佳/最危险的一句话回放
- 现实行动建议（一句话）

## 四、技术架构

### 技术栈
- 前端：React（纯前端SPA）
- AI模型：阶跃星辰 API（OpenAI兼容格式）
- 部署：本地开发服务器 / Vercel（评委试玩用）

### API调用设计

**核心原则：单次调用返回全部内容，前端零计算。**

每轮对话只发一次API请求，返回结构化JSON，包含NPC回复、数值变化、下一轮卡牌、游戏状态。

**请求结构**：
```
POST /v1/chat/completions
{
  "model": "step-2-16k",  // 根据实际可用模型调整
  "messages": [
    { "role": "system", "content": "<<完整系统prompt>>" },
    { "role": "user", "content": "<<对话历史 + 用户本轮出牌内容 + 当前数值状态>>" }
  ],
  "response_format": { "type": "json_object" },
  "temperature": 0.7
}
```

### 前端解析逻辑
拿到JSON后直接渲染，不做二次计算。数值条变化用CSS transition动画。气泡逐条setTimeout(500ms)弹出。卡牌区在NPC说完话后展开。

### 容错机制
- JSON解析失败 → 自动重试一次（同样的请求）
- 连续两次失败 → 展示"网络波动，请稍等"，5秒后重试
- 角色漂移兜底 → 每轮user message头部带角色锚定重复段

## 五、Prompt架构

### 系统Prompt结构（以MVP场景1为例）

```
你是一个社交训练游戏的AI引擎。你的任务是扮演NPC角色与用户进行对话博弈。

## 场景设定
- 场景：年终review，一对一谈话
- 地点：公司会议室
- 背景：用户是入职两年的设计师，月薪8K，今年绩效优秀。老板约他谈话，实际目的是压住涨薪。

## 你扮演的角色
- 姓名：张总
- 身份：部门总监，用户的直属上级
- 性格类型：笑面虎
- 行为模式：表面温和关怀，实际每句话都在施压。善用"我也是为你好""公司不容易"等话术。不会主动提涨薪，如果用户提出会先打太极再压价。
- 底层目标：把涨薪幅度压到5%以内，最好是0。

## 不可违反的规则（每轮必须遵守）
1. 绝不主动同意涨薪，除非用户连续3轮以上用数据和逻辑施压且主动权>80
2. 绝不打破角色，不能变成暖心导师、不能直接告诉用户该怎么做
3. 不使用暴力威胁或违法言论
4. 每句话都必须推进谈话，不能原地转圈说废话
5. 根据用户的出牌类型调整策略：用户硬刚则你软化但不让步，用户软弱则你加大施压

## 用户人设
{user_persona}
（如用户跳过人设配置，此处填入：入职两年普通员工，性格中等偏内向，有能力但不善于表达诉求）

## 数值系统规则
- 主动权(initiative)：0-100，初始值40（老板先手优势）
  - 用户用数据说话 +10~15，用户被动接受 -10~15，用户硬刚 +5但耐心-15
- 对方耐心(patience)：0-100，初始值80
  - 用户合理表达 -5，用户硬刚 -15，用户打太极 -3，归零=谈崩=D级结局
- 目标达成率(goal_progress)：0-100，初始值10
  - 每次有效推进涨薪话题 +8~15，被转移话题 -5，被PUA接受 -10

## 输出格式
严格按以下JSON格式输出，不要有任何额外文字：
{示例JSON，与技术架构章节中的返回结构一致}

## 重要
- 每轮的4张卡牌必须风格明确区分，不能4张都是合理回复
- C类（学生思维）卡牌必须是明显会吃亏的选项，让用户在对比中学习
- 数值变化要符合逻辑，不能随机跳动
- 对话推进要有节奏感：前3轮试探，中间4轮拉锯，最后3轮摊牌
```

```
{用户选择的卡牌full_text}

```

### 多人场景Prompt补充段（场景2：同事甩锅）

在系统prompt中增加第二个NPC定义：
```
## 你同时扮演的角色（按顺序发言）

### 角色1：李明（甩锅同事）
- 身份：同部门平级同事
- 性格类型：暴躁易怒
- 行为模式：先发制人，把项目延期的责任全推到用户头上。语气急躁，爱打断人，喜欢用"我早就说过"开头。
- 底层目标：让领导相信是用户的问题，保住自己的绩效。

### 角色2：王总（观望领导）
- 身份：部门经理，两人共同上级
- 性格类型：沉着冷静
- 行为模式：不会直接站队，主要通过提问来判断真相。"你怎么看？""有证据吗？"
- 底层目标：搞清楚真相，同时不想让团队氛围太僵。

## 多人发言顺序规则
- 每轮先由事件触发方（李明）发言，再由王总简短回应或提问，最后才是用户的出牌机会
- npc_responses数组中按此顺序排列
- 卡牌需标注对话对象："对李明说" 或 "对王总说" 或 "对两人说"
```

## 六、团队分工与时间线

### 角色定义

| 角色 | 职责 | 核心产出 |
|------|------|---------|
| P1 前端主力 | UI开发、组件搭建、API对接 | 可交互的完整五屏应用 |
| P2 Prompt工程师 | Prompt编写、API调试、角色调优 | 稳定输出合规JSON的prompt套件 |
| P3 内容策划 | 剧本框架、对话节奏、评分规则、文案 | 完整的场景剧本和评分逻辑 |
| P4 视觉+兜底 | UI设计、素材、测试、Demo保障 | 视觉方案+录屏备份+bug清单 |

### 时间线

#### 第一小时（0:00-1:00）：地基

**P1**：
- 初始化React项目，搭建路由（五屏框架）
- 完成第一屏（场景三级标签选择页）基本UI
- 完成第二屏（人设问答页）基本UI
- 约定好与P2的API接口数据结构

**P2**：
- 完成场景1（1v1老板PUA）的完整系统prompt
- 在API调试工具中跑通：输入一句话 → 输出合规JSON
- 重点调试：JSON格式稳定性、角色一致性
- 与P3结对确认角色行为边界

**P3**（前半小时与P2结对）：
- 定义老板角色的行为边界清单：什么条件下让步/加压/转移话题/摊牌
- 定义10轮对话的节奏曲线：轮1-3试探、轮4-7拉锯、轮8-10摊牌
- 编写场景1的briefing文案
- 编写5个结局等级的判定条件和点评文案

**P4**：
- 确定视觉方案（配色、卡牌样式、数值条样式）
- 找/制作素材：场景背景图、NPC头像、卡牌底图
- 设计数值条的动效方案（CSS transition参数）
- 将设计token（颜色值、字号、间距）整理给P1

#### 第二小时（1:00-2:00）：跑通

**P1**：
- 完成第三屏（对话主界面）核心布局：气泡区+卡牌区+数值条
- 接入P2的API，跑通单轮循环：NPC说话→出牌→NPC回应→数值变化
- 完成第四屏（结局页）基本布局

**P2**：
- 与P1联调API，解决实际返回格式问题
- 调试角色防漂移：连续跑10轮，检查角色是否出戏

**P3**：
- 编写其余场景的briefing文案（不可玩场景的展示内容）

**P4**：

## 九、Demo话术参考（1分钟Pitch）

> "别把天聊死"是一个AI社交训练沙盘。
>
> 我们发现一个问题：很多人在职场、家庭、社交中遇到高压对话时不知道怎么开口。市面上的产品要么给你话术模板——但真到了现场你还是说不出口，要么是AI陪聊——但没有目标感和反馈。
>
> 我们的做法是把它做成剧本杀：你选一个场景，比如老板PUA你压薪，系统给你一个真实的NPC对手，你通过出牌来推进对话，有数值博弈，有结局评分。不是教你说什么，是让你练怎么说。
>
> 现在请评委老师试玩一局。
